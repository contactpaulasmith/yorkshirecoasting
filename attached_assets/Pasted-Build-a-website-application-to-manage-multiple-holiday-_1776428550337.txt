Build a website application to manage multiple holiday let properties and attract potential guests to book – this solution will be built in phases. 

The brand should feel warm, welcoming, high-quality, and relaxing — like a premium home away from home.
The website should include:
1.	A homepage with:
•	A large hero image (cosy, stylish holiday property)
•	Clear call-to-action buttons (e.g. “View Properties”, “Book Now”)
2.	A properties page:
•	Grid layout of properties
•	Each property should include:
o	Image
o	Name
o	Short description
o	Price per night
o	“View Details” button
3.	Property detail page:
•	Image gallery
•	Full description
•	Key features (e.g. location, number of guests, amenities)
•	Booking link or enquiry form
4.	Contact section:
•	Simple enquiry form (name, email, message)
•	Optional phone number and location
5.	FAQs
1.	Location
2.	Ability to add other content
6.	Design style:
•	Warm colour palette (beige, soft browns, off-white)
•	Clean, modern layout
•	High-end but cosy feel (like Airbnb premium listings)
•	Use plenty of whitespace and soft shadows
6.	Technical requirements:
•	Use React (or Next.js if possible)
•	Mobile responsive design
•	Clean, well-structured code
•	Use reusable components
•	Include placeholder images and dummy data
Make the site feel premium, calm, and inviting — not corporate or overly minimal.

Future phases:
The website purpose in the future is : to promote multiple holiday let accommodations to grow bookings and minimize the effort to welcome and communicate with guests who have booked direct and contractors who assist in cleaning and other maintenance services
There are 3 primary users
1.	Potential Guests who will book
2.	Administrators who will contribute content, add properties, engage with guests and maintenance and cleaning contractors
3.	Indirectly - Cleaning and other contractors who will receive and send communications in relation to the properties and booking
Each user type will see different aspects of the website
