The Netlify deploy errored, with the following guidance provided:

Relevant error lines
- [line 85](#L85) shows Vite failed to load the project config.
- [line 87](#L87) is the actual failure: "Error: PORT environment variable is required but was not provided."
- [line 88](#L88) shows the error is thrown while loading modules/config during the build.

Diagnosis
- Error type: runtime exception thrown while loading Vite config (build-time crash).
- Cause: your Vite (or another config file loaded by Vite) requires process.env.PORT and throws when it's missing. That requirement runs during config loading, so the build aborts on Netlify because PORT is not set in the build environment.

Solution (two safe options — pick one)

Option A — Set PORT in Netlify build environment (quick)
1. In the Netlify UI go to Site settings → Build & deploy → Environment → Environment variables, add:
   - Key: PORT
   - Value: 5173
2. Re-run the build.

Alternatively add the env var in netlify.toml:
```toml
[build.environment]
  PORT = "5173"
```

Option B — Make the config tolerant (recommended long-term)
Edit the Vite config (or whatever file enforces PORT) in artifacts/mockup-sandbox so it doesn't throw during build and uses a sensible default. For example, in artifacts/mockup-sandbox/vite.config.ts (or .js) change the strict check:

Before (problematic — throws when PORT is missing):
```ts
if (!process.env.PORT) throw new Error('PORT environment variable is required');
```

After (use default and only require for dev if needed):
```ts
const PORT = process.env.PORT ?? '5173';

export default defineConfig(({ command }) => {
  return {
    server: {
      port: Number(PORT),
    },
    // other config...
  };
});
```

Or only require PORT for the dev server:
```ts
export default defineConfig(({ command }) => {
  if (command === 'serve' && !process.env.PORT) {
    throw new Error('PORT environment variable is required for local dev');
  }
  // use default for build:
  const port = process.env.PORT ?? '5173';
  return { server: { port: Number(port) } };
});
```

Notes
- The error originates while loading the project config (see [line 85](#L85) and [line 87](#L87)), so the fix must prevent the throw during config load (either by providing PORT in Netlify or by making the config tolerate a missing PORT).
- If you edit files, commit them and push before re-running the Netlify build.
